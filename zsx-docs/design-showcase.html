一、产品总览

Multi-Layer Mindmap Blog —— 将思维导图的层级结构、无限下钻的聚焦能力、以及 Markdown 富文本笔记三者融合，构建一个树形知识库系统。

核心定位

将一本书（或课程/知识体系）以树形思维导图的方式组织，每个节点附带 Markdown 正文，最终导出为独立 HTML 文件分享。

无限下钻 Markdown正文 导出HTML 家族图谱

竞品空白

目前 GitHub 上没有任何开源项目同时具备「下钻」+「节点 Markdown 正文」+「纯 Web 无限画布」这三个核心能力。

基于 SimpleMindMap（12k★, MIT）二次开发，预估 MVP 2-4 周。

二、三套原型方案

方案 A · 经典双栏 桌面端推荐

左侧画布 + 右侧 Markdown 正文面板。适合编辑与阅读并重的场景。

方案 B · 沉浸阅读 导出HTML默认

暗色背景，目录卡片列表 + 章节阅读视图。类似电子书阅读器。

方案 C · 混合探索

左侧大纲树 + 右侧 Markdown 正文。最省空间的布局。

三、双模交互架构

同一份 Mindmap 数据，画布模式与阅读模式无缝切换。Ctrl+R 切换，Esc 返回。

四、节点类型系统

7种节点类型，帮助用户快速识别节点性质。支持手动设置或根据内容自动推断。

五、家族关系图谱（扩展方向）

思维导图 ≠ 家族图谱。家族图谱是「图」而非「树」，需要夫妻容器、世代分层、跨图谱跳转三种机制。
